## Query Method Patterns

| Method Signature | Query Type | Example |
|------------------|-----------|---------|
| findByName(String) | Generated query | findByName("John") |
| findByEmail(String) | Generated query | findByEmail("john@ex.com") |
| findById(Long) | JPA method | findById(1L) |
| findAll() | JPA method | Get all records |

## Repository Query Methods

```java
// Custom query methods (auto-generated by Spring)
List<User> findByName(String name);        // WHERE name = ?
User findByEmail(String email);            // WHERE email = ?

// JPA standard methods
Optional<User> findById(Long id);
List<User> findAll();
long count();
void deleteById(Long id);
```

## Test Data Setup and Assertions

```java
// Setup
User user1 = new User(null, "John");
entityManager.persistAndFlush(user1);

// Query
List<User> results = userRepository.findByName("John");

// Assert
assertEquals(1, results.size());
assertEquals("John", results.get(0).getName());
```

## Query Result Types

| Return Type | Method | Example |
|------------|--------|---------|
| List<User> | Multiple results | findByName() returns 2+ users |
| User | Single result | findByEmail() returns 1 user |
| Optional<User> | Nullable single | findById() may be empty |
| Long | Count | count() returns total |
| Boolean | Exists | existsById() returns true/false |

## Best Practices

1. ✓ Test single and multiple results
2. ✓ Test empty/not-found scenarios
3. ✓ Verify query accuracy
4. ✓ Test edge cases
5. ✓ Validate result ordering
